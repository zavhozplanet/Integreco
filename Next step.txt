# Integreco Project Summary: Vision and Roadmap

## 1. Global Concept and Philosophy
**Integreco** is an interactive, offline-first environment for visual thinking and building complex relationship graphs.

* **Goal**: Create an "exocortex" (second brain) that not only stores human thoughts but also acts as a bridge for transmitting conceptual vision to AI models.
* **Fractality**: The project is built on the idea of ​​nesting—nodes can be other maps, people, or objects.

## 2. Evolution Roadmap
[cite_start]The project's development proceeds through successive stages without radically reworking the existing code[cite: 1, 2]:
1. [cite_start]**TypeScript + Vite**: Transition to a modern development environment and a typed node schema (current stage)[cite: 1, 2, 3].
2. [cite_start]**AI Export**: Graph serialization into semantic JSON-LD or a custom contextual format for direct transfer of meanings in LLM (Claude, GPT)[cite: 4].
3. [cite_start]**3D View**: A separate graph visualization mode in 3D space[cite: 5].
4. [cite_start]**Desktop App**: Electron wrapper for direct access to the file system without browser limitations[cite: 5].
5. [cite_start]**Social/Sync Layer**: Using lightweight solutions (Supabase/PocketBase) for cloud sync and social features[cite: 6].
6. **Obsidian Plugin**: Full integration with the Obsidian ecosystem.

## 3. Key Architectural Decisions

### "Knowledge as a File System" (Obsidian-like) Principle
Instead of storing data only within a closed graph, the project moves to a **Markdown Vault** structure:
* **Node = File**: Information in nodes is stored as `.md` files.
* **Hierarchy**: Parental relationships in the graph are displayed as a folder structure.
* **Semantic Links**: Relationships between nodes (`isLink: true`) are written within texts as wiki links `[[Node]]`. * **Two-way communication**: Changes in the file system (via external editors) should be reflected in the map, and vice versa.

### Agent Memory (LLM OS)
Instead of the classic RAG (search through vectorized pieces of text), an agent memory architecture (based on Andrej Karpathy) is implemented:
* **Core Memory**: The current context of the open canvas.
* **Archival Memory**: A local database of Markdown files.
* **Tools**: The AI ​​gains the ability to "walk" the file system through system calls (reading/writing nodes).

### Self-Learning Hybrid (Hermes Agent)
Integration with open-source agents (such as Hermes from Nous Research) to create a self-improving mechanism:
* **Learning Loop**: The agent analyzes the user's thinking patterns and generates "Skill Documents" in Markdown format. * **Background Process**: The agent runs in the background, analyzing changes in the knowledge base and suggesting new connections.

## 4. UI/UX and Security Mechanics

### Change Verification ("Ghost Nodes")
To prevent uncontrolled graph changes by the AI ​​agent, a confirmation system is used:
* **Ghost Nodes**: AI suggestions (new nodes or connections) are displayed on the canvas as translucent or dotted lines.
* **Activity Indicator**: Nodes that the AI ​​(or another user) wants to discuss begin to gently **pulse**.
* **Dialog Box**: Clicking on a pulsing node opens a contextual chat for discussing meanings.

### Three-Level Security
1. **Snapshot Backups**: Automatic file backups before AI intervention. 2. **AI Inbox**: All new agent proposals are first written to the isolated `_AI_Proposals/` folder.
3. **Visual Approval**: Nodes become "real" (transferred to the main database) only after user approval on the canvas.

## 5. Social Network as a Fractal Graph
* **People as Nodes**: In the global social map, nodes are people.
* **Dive**: Clicking on a person node allows you to "dive" inside and see their public mind map.
* **AI Facilitator**: In collaborative discussions, AI acts as a mediator, helping different points of view complement each other to form a coherent picture.

## 6. Fundamental Dogmas (Constraints)
* **Root Node**: The `nodes[0]` node is always the root and cannot be deleted. * **Surgical Edits**: Code editing is limited to specific areas, without completely rewriting modules.
* **Offline-first**: Full functionality without internet connection via the File System Access API and IndexedDB.
* **Geometry**: Node sizes are calculated only via `canvas.measureText()`, without using `getBoundingClientRect()`.
