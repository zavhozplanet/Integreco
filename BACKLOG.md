## 1. 🔴 КРИТИЧЕСКИЕ И СРЕДНИЕ БАГИ (Текущий фокус)

### 🔴 КРАСНАЯ ЗОНА (КРИТИЧЕСКИЕ БАГИ И ПРИОРИТЕТЫ)


### Группы и Линии

- [ ] **Зона ресайза группы:** Непропорционально смещена наружу и слишком большая зона хвата для ресайза группы. Надо сделать чтобы центр этой зоны четко совпадал с центром изгиба угла рамки группы, а зона хвата была в два раза меньше.
- [ ] **Кнопки слоев в меню группы:** Убрать из меню группы кнопки "на передний план" и "на задний план" (логика слоев группы уже определена и зафиксирована ранее).

### Логика связок (Edges)
- [ ] **Баг сворачивания связок:** Некоторые связки (связующие линии) сами разворачиваются при сворачивании/разворачивании других связок. Сворачивание конкретной связки влияет на отображение соседней связки (она может сворачиваться тоже). Надо чтобы сворачивание конкретной связки сворачивало только эту связку и не влияло ни на какие другие связки, линии и объекты.
- [ ] **Стрелки на связках (пунктир):** При добавлении стрелки для связки, которая по умолчанию пунктирная и без стрелок, стрелка не появляется. Если изменить тип на прямой - стрелка появляется и остаётся при переключении обратно на пунктир. Нужно чтобы стрелка на связке появлялась (и/или настройка применялась) сразу при её добавлении, при любых условиях.

### UI / Интерфейс

- [ ] **Адаптивность всех меню:** Меню настроек, ПКМ-меню объектов/линий, меню холста (любые меню) при близости к периферии экрана выходят за пределы видимой части приложения. Надо чтобы меню отображалось адаптивно, выбирая место так, чтобы его было видно полностью.
- [ ] **Закрытие меню вставки в разрыв:** После клика на "+" и выбора типа объекта в меню линии, объект вставляется в линию, но меню не исчезает. Надо чтобы выбор объекта одновременно вставлял объект в линию и закрывал меню.


### ✅ ЗАВЕРШЕНО
- [x] **Защита Root-объекта:** Запрещено случайное удаление корня при массовом выделении. Добавлена кнопка удаления в индивидуальном меню, активная только если у корня нет потомков. Если root был отделён путём удаления линии, статус root автоматически переходит к его бывшему дочернему объекту.
- [x] **Восстановление ПКМ после удаления root:** Исправлены падения UI при отсутствии корневого узла.
- [x] **Кнопка «Центрировать» (Double-tap):** Реализовано центрирование на Root при двойном клике на кнопку ⊙.
- [x] **Стабилизация зума/пана:** Исправлены ошибки при расчёте координат на больших уровнях зума.

## 2. 🟡 СТАБИЛИЗАЦИЯ И UX УЛУЧШЕНИЯ

### Текст и ввод
- [ ] **Адаптивный текст при прямом вводе:** При введении текста во время добавления объекта (прямо в текстовом поле узла) он не адаптивный. При темном фоне холста цвет текста остается черным и плохо виден.
- [ ] **Умолчания для вставляемых узлов (копипаст):** При копировании объекта и вставке его через меню как нового дочернего, он добавляется с умолчаниями нового приложения, а не с текущими умолчаниями настроек данного типа объектов. Надо чтобы объекты копирования/вставки добавлялись с применением текущих умолчаний данного типа объектов.
- [ ] **Приоритет направления создания нового объекта (+):** Т.е. если есть свободное место по направлению от кнопки-плюса (от левой - слева, правой - справа, верхней - вверху, нижней - внизу), то новый узел создаётся в этом направлении. Если свободного места по направлению нет - применяются действующие правила.



### 🟡 ЖЕЛТАЯ ЗОНА (СРЕДНИЙ ПРИОРИТЕТ)
- [ ] Очистка корзины: Добавить кнопку «Очистить всё» в модальное окно корзины.
- [ ] Групповой фон и Групповой заголовок (Разделение): В меню настройки группы добавить раздельные настройки для opacity фона и opacity заголовка (сейчас могут быть связаны).

## 3. 🟢 НОВЫЕ ФИЧИ (ГРУППА 3 И 4)

### Линии и фиксация
- [ ] **Локальная фиксация линии на объекте (кроме групп и multi):** Переключатель в меню линии "зафиксировать". Закрепляет данную линию на ближайшей точке с кнопкой-плюсом.
- [ ] **Произвольная локальная фиксация линии на группе:** Тот же переключатель, но для группы фиксирует исходящую и входящую линии в том месте, где она в данный момент входит или выходит из группы. Функция НЕ касается линий, пересекающих рамку группы (соединение объектов в группе и вне её).
- [ ] **Передвигание/переподключение крепления линии к группе:** С такой же логикой, как на мультиузле (drag to reconnect/move on border).

### Оформление и текст
- [ ] **Меню форматирования текста:** Добавить меню со стандартным набором инструментов форматирования текста при ПКМ на текстовом поле (размер шрифта, выбор шрифта, цвет, выравнивание текста, и т.д.).
- [ ] **Больше рамок в меню настроек:** Добавить в блок выбора рамок больше видов рамок (посмотри примеры сервисов Майнд-карт), добавить полосу прокрутки для их просмотра и выбора. Текущие три вида рамок оставить самыми первыми, вверху набора.

### Интерфейс объектов
- [ ] **Применить к выделенным:** Функция в меню объектов. При открытии меню одного из нескольких выделенных объектов, настройка в нём применяется ко всем выделенным объектам.
- [ ] **ПКМ-меню для кнопки-плюса:** Добавить для кнопки-плюса с выбором типа добавляемого объекта.

## 4. 🔵 БУДУЩИЕ КРУПНЫЕ ОБНОВЛЕНИЯ (КАТАЛОГ И ОБЛАКО)

### Каталог и Структура
- [ ] **Каталог карт в главном меню:** Для открытия по одному клику (чтобы не заходить в рабочую папку через "открыть из папки"). Карту при создании в новой вкладке (по кнопке "+" в главном меню) сохранять в рабочей папке.
- [ ] **Авто-имя файла карты:** При создании заголовка для первого объекта новой карты в новой вкладке, текст заголовка автоматически используется для имени файла карты в рабочей папке и каталоге.
- [ ] **Окно каталога:** Открывается в полноразмерном окне (в том же состоянии, что и основное окно), в виде плиток или горизонтальных карточек с превью-эскизами карт, с кнопками выбора отображения плитка/список. На карточке кнопки: поделиться, скачать (подменю), удалить в корзину.
- [ ] **Логика открытия карты из каталога:** Двойной клик на карточку открывает карту в новой вкладке. Если открыта вкладка с пустым холстом с плюсом (карта не начата), клик по карточке открывает карту в текущей вкладке.
- [ ] **Кнопка "Открыть объект как карту":** В меню объектов. Создаёт новую карту в новой вкладке (и в папке, и в каталоге) с данным объектом как корневым.
